# 📘 Assignment: Testing and Refactoring Python Code

## 🎯 Objective

In this assignment, you will write tests for existing Python code and then refactor that code to improve readability and maintainability. You will practice identifying bugs, protecting behavior with tests, and making safe improvements.

## 📝 Tasks

### 🛠️	Write Baseline Tests

#### Description
Use the provided starter code to create tests that describe the current and expected behavior of each function.

#### Requirements
Completed program should:

- Create a test file named `test_starter_code.py` using pytest.
- Add tests for price totals, discount logic, and student grade calculation.
- Include at least one edge-case test (such as empty input, boundary values, or invalid values).


### 🛠️	Fix Bugs and Make Tests Pass

#### Description
Run your tests, identify failing cases, and fix implementation bugs in the starter code.

#### Requirements
Completed program should:

- Update the functions in `starter-code.py` so failing tests pass.
- Keep function names and parameters unchanged.
- Ensure all tests pass consistently when run multiple times.


### 🛠️	Refactor for Clean Code

#### Description
Improve the internal structure of the code while preserving behavior verified by your tests.

#### Requirements
Completed program should:

- Refactor at least two functions to improve naming, readability, or duplication.
- Keep behavior unchanged, confirmed by passing tests.
- Add concise comments only where logic is not immediately obvious.
